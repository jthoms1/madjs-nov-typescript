Preact's Goals

  • Performance: Render quickly & efficiently
  • Size: Small size, lightweight (approximately 3.5kb)
  • Efficiency: Effective memory usage (recycling, avoiding GC thrash)
  • Understandability: Understanding the codebase should take no more than a few hours
  • Compatibility: Preact aims to be largely compatible with the React API. preact-compat attempts to achieve as much compatibility with React as possible.

How is it the same as React?

  • Virtual Dom
  • Component Composition
  • JSX + hyperscript based
  • State and Props

How is it different than React?

  • PropType Validation (preact-compat)
  • Synthetic Events (browser events only)
  • "className" is now "class" in JSX (yay!!!)
  • linkState for binding values to state updates

Today's tiny frameworks

  • Leveraging the browser platform (ie9+)
  • Drop components on to websites or build entire apps
  • All about the Vdom and hyperscript
